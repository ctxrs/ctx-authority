# Core beliefs

1. Agents should get capabilities, not durable secrets.
2. Local mode must be useful without signup.
3. Fake providers come before real providers.
4. Every meaningful action should produce an audit event.
5. Executed actions should produce signed receipts.
6. Approval must bind to an exact canonical action hash.
7. Policy ambiguity should fail closed.
8. The repository should be legible to coding agents.
9. Public code is not the moat; trust, distribution, hosted operations, and
   receipt adoption are the moat.
10. The product should start narrow and earn the right to become broader.
